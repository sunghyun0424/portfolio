# 가비아 DNS로 `sunghyun.my` → Vercel 연결

이 포트폴리오는 **Vercel**에 배포되어 있고, 저장소의 `vercel.json`에는 **`www.sunghyun.my` → `https://sunghyun.my`** 영구 리다이렉트가 들어 있습니다.  
아래는 **도메인을 가비아에 두고**, DNS만 가비아에서 수정하는 방법입니다.

---

## 1. Vercel에서 도메인 등록 (먼저)

1. [Vercel](https://vercel.com) → 포트폴리오 **프로젝트** 선택  
2. **Settings → Domains**  
3. **`sunghyun.my`** 입력 후 추가  
4. 같은 화면에서 **`www.sunghyun.my`** 도 추가(선택이지만 권장)  
5. Vercel이 표시하는 **권장 DNS 레코드**(A / CNAME 등)를 캡처해 두세요.  
   - Vercel은 정책·시기에 따라 IP나 호스트명을 바꿀 수 있으므로, **최종 값은 이 화면에 나온 것이 기준**입니다.

---

## 2. 가비아에서 DNS 레코드 설정

1. [My가비아](https://my.gabia.com) 로그인  
2. **서비스 관리** → **도메인** → **`sunghyun.my`** 선택  
3. **DNS 관리** 또는 **DNS 정보** / **DNS 설정**(메뉴 이름은 UI에 따라 조금 다를 수 있음)으로 이동  
4. **네임서버가 가비아 기본(가비아 DNS)** 인지 확인합니다.  
   - 다른 곳(Cloudflare 등)으로 네임서버를 넘겼다면, **그 서비스 쪽**에서 같은 내용으로 레코드를 넣어야 합니다.

### 루트 도메인 (`sunghyun.my`)

루트에는 보통 **CNAME을 쓸 수 없는** 경우가 많아, Vercel 안내가 **A 레코드**이면 아래처럼 넣습니다.

| 구분 | 예시 (가비아 입력 시 참고) |
|------|---------------------------|
| 타입 | **A** |
| 호스트(호스트명) | **`@`** 또는 **비움** / **도메인명만**(가비아 화면 안내에 따름) |
| 값(IP) | Vercel Domains에 표시된 IP (문서상 예: `76.76.21.21` — **반드시 Vercel 화면 값 확인**) |
| TTL | 기본값 또는 600초 전후 |

Vercel이 **ALIAS / ANAME** 사용을 안내하면, 가비아 DNS 화면에 해당 타입이 있는지 확인 후 같은 대상으로 설정합니다.

### `www` 서브도메인 (`www.sunghyun.my`)

| 구분 | 예시 |
|------|------|
| 타입 | **CNAME** |
| 호스트 | **`www`** |
| 값(대상) | Vercel이 안내하는 호스트 (보통 `cname.vercel-dns.com` 계열 — **Vercel 화면 그대로**) |
| 끝의 `.` | 가비아는 **FQDN 끝에 마침표(`.`)** 를 요구하는 경우가 있어, 저장이 안 되면 `cname.vercel-dns.com.` 형태로 시도 |

---

## 3. 이메일·기존 레코드

- **MX**(메일), **TXT**(인증·SPF 등)는 **삭제하지 마세요.**  
- 루트(`@`)에 이미 **A 레코드**가 다른 용도로 있으면, **한 호스트에 A가 중복**되면 안 되므로 Vercel 연결 전에 용도를 정리해야 합니다.

---

## 4. 적용 확인

- 저장 후 **수 분 ~ 최대 48시간** 전파될 수 있습니다.  
- Vercel **Domains**에서 “Valid” 같은 상태로 바뀌는지 확인합니다.  
- 터미널 예시: `dig sunghyun.my A +short` / `dig www.sunghyun.my CNAME +short`  
- 브라우저에서 `https://sunghyun.my` 접속 후 SSL(자물쇠)이 정상인지 확인합니다.

---

## 5. 이 저장소와 맞는 설정

- 공식 URL은 **`https://sunghyun.my`** 기준으로 `index.html`에 canonical·OG URL이 잡혀 있습니다.  
- `www`로 들어오면 **`https://sunghyun.my`** 로 리다이렉트되도록 `vercel.json`에 설정되어 있습니다.

문제가 계속되면 Vercel Domains 화면의 **에러 메시지**와 가비아 **현재 DNS 레코드 목록**을 함께 보면 원인 찾기가 쉽습니다.
